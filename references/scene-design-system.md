# 场景设计系统

适用于所有领域的 7 场景通用模板 + 设计令牌 + 时序公式。

## 默认设计令牌

写在 HTML `<style>` 的 `:root` 中，可根据领域调整：

```css
:root {
  --blue: #1A56DB;       /* 主色 */
  --dark: #0F1D3C;       /* 深色背景 */
  --accent: #3ABFF8;     /* 浅蓝高光 */
  --white: #FFFFFF;      /* 主文字 */
  --gray: #94A3B8;       /* 辅助文字 */
  --card-bg: rgba(255,255,255,0.06);
  --card-border: rgba(255,255,255,0.12);
}
```

字体：`"PingFang SC", "Noto Sans SC", "Microsoft YaHei", sans-serif`

### 领域调色速查

| 领域 | 主色 | 背景基调 | 高光 |
|---|---|---|---|
| 企业/科技 | #1A56DB | #0F1D3C 深蓝 | #3ABFF8 |
| 电商/消费 | 品牌色 | 品牌深色 | 品牌亮色 |
| 教育 | #2563EB 或暖色 | #1E293B | #60A5FA |
| 资讯/新闻 | #DC2626 | #0C0C0C | #EF4444 |
| 医疗/健康 | #059669 | #0F1D20 | #34D399 |

## 7场景通用模板

```
0    S1    S2     S3    S4a   S4b   S4c       S5          S6    S7
├────┼────┼──────┼──────┼─────┼─────┼─────────┼──────────┼─────┼────┤
封面  痛点  方案  功能1  功能2  功能3    优势/特性    数据/证据   CTA
```

### 场景详表

| # | 名称 | 时长 | 布局 | 内容 |
|---|---|---|---|---|
| 1 | 封面 | 4-10s | 左标题右配图 | 主标题+副标题+标签行+封面图 |
| 2 | 痛点/背景 | 5-11s | 2×2 卡片网格 | 4个痛点或背景信息卡片 |
| 3 | 方案/定位 | 4-11s | 居中大字 | 核心主张+3个价值badge |
| 4a | 功能/内容A | 5-12s | 左列表右图 | 3条要点+右侧示意 |
| 4b | 功能/内容B | 5-13s | 同上 | 同上，内容换组 |
| 4c | 功能/内容C | 5-10s | 双卡并排 | 2个重点内容 |
| 5 | 优势/特性 | 7-13s | 3列卡片 | 6条优势（2×3布局） |
| 6 | 数据/证据 | 5-12s | 居中大字 | 4个数字+countUp动画 |
| 7 | CTA | 5-14s | 居中 | 标语+联系方式 |

### 场景数量可调

- **短版（30-45s）**：封面 → 痛点 → 方案 → 功能(合并4a+4b) → 数据 → CTA（6场景）
- **标准版（60-70s）**：完整 7 场景
- **长版（90-110s）**：完整 7 场景 + 每个场景匹配旁白时长

## 场景时序公式

### 无旁白版
```js
// 每个场景的 data-start 和 data-duration 从脚本直接取
// 参考值（45s 总长）
data-start="0"  data-duration="4"   // 封面
data-start="4"  data-duration="5"   // 痛点
data-start="9"  data-duration="4"   // 方案
data-start="13" data-duration="5"   // 功能A
data-start="18" data-duration="5"   // 功能B
data-start="23" data-duration="5"   // 功能C
data-start="28" data-duration="7"   // 优势
data-start="35" data-duration="5"   // 数据
data-start="40" data-duration="5"   // CTA
```

### 有旁白版
每个场景的 `data-duration` = 旁白时长 + 0.5s 缓冲。
`data-start` = 前一个场景的 `data-start + data-duration`。

先跑完所有 TTS，用 `ffprobe` 获取每段精确时长，据此计算时序。

## 配音文案公式

每场景旁白字数估算（中文 Edge TTS 约 4字/秒）：
- 封面：20-30字 → 5-7s
- 痛点/背景：35-50字 → 9-12s
- 方案：25-40字 → 6-10s
- 功能A/B/C：各 30-45字 → 8-12s
- 优势：40-55字 → 10-13s
- 数据：30-45字 → 8-12s
- CTA：40-55字 → 10-13s
- **总计**：约 250-380字 → 60-95s 总旁白

## 领域适配指南

### 企业方案（标准模板）
标准 7 场景，强调数据+案例+CTA联系。痛点卡片列出行业问题，功能分 3 组展示。

### 电商产品
场景调整：封面→痛点→产品展示×3（外观/功能/对比）→口碑数据→购买CTA
图片占比大，产品图做封面和功能展示的右列配图。

### 教育课程
场景调整：封面→学习痛点→课程方案→内容大纲×2→学习效果→报名CTA
文字为主，卡片网格多。可去掉"数据效果"场景或替换为"适合人群"。

### 资讯报道
场景调整：封面→事件背景→关键信息×2→多方观点→展望CTA
快节奏，每场景 3-5s。痛点和方案场景可合并。
